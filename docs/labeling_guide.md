# 리뷰 라벨링 가이드

## 목적
AI 모델의 정확도를 측정하기 위한 Ground Truth 데이터 생성

## 라벨링 규칙

### 1. 주요 카테고리 (Primary Issue)

리뷰를 읽고 **가장 핵심적인 문제 1가지**를 선택하세요.

#### 카테고리 목록:

| 카테고리 | 설명 | 예시 |
|---------|------|------|
| `delivery_delay` | 배송이 늦음 | "2주 걸렸어요", "배송이 너무 느려요" |
| `wrong_item` | 잘못된 상품 수령 | "다른 색상 왔어요", "주문한 게 아니에요" |
| `poor_quality` | 품질 불량 | "재질이 싸구려", "금방 고장남" |
| `damaged_packaging` | 포장/상품 파손 | "박스가 찢어짐", "깨진 채로 도착" |
| `size_issue` | 사이즈 문제 | "생각보다 작아요", "사이즈가 안 맞아요" |
| `missing_parts` | 부품 누락 | "설명서가 없어요", "부속품이 빠졌어요" |
| `not_as_described` | 설명과 다름 | "사진과 달라요", "기능이 없어요" |
| `customer_service` | 고객 서비스 문제 | "환불 안 해줘요", "연락이 안 돼요" |
| `price_issue` | 가격 문제 | "너무 비싸요", "가성비 안 좋아요" |
| `other` | 기타 | 위 카테고리에 해당 안 됨 |

### 2. 라벨링 기준

#### ✅ 좋은 예:
- **핵심 문제에 집중**: 여러 문제가 있어도 가장 중요한 것 1개
- **객관적 판단**: 리뷰 내용 기반으로만 판단
- **일관성**: 비슷한 케이스는 같은 라벨

#### ❌ 나쁜 예:
- 주관적 해석 추가
- 리뷰에 없는 내용 추측
- 매번 다른 기준 적용

### 3. 복합 문제 처리

리뷰에 2개 이상의 문제가 있을 때:

**규칙**: 문장/단어 수가 많은 쪽을 선택

**예시**:
```
"배송이 조금 늦었는데, 상품 품질이 정말 최악이에요. 재질도 싸구려고 금방 망가졌어요."

→ poor_quality (품질 언급이 더 많음)
```

```
"포장이 찢어져서 도착했고, 배송도 3일 늦었어요."

→ damaged_packaging (먼저 언급 + 동등하면 첫 문제 선택)
```

### 4. 애매한 케이스

#### Case 1: 너무 짧은 리뷰
```
"별로예요"
→ other
```

#### Case 2: 긍정+부정 혼재
```
"배송은 빨랐는데 품질이 별로예요"
→ poor_quality (부정적 부분 선택)
```

#### Case 3: 영어/포르투갈어
```
문맥으로 판단하거나 번역기 사용
```

### 5. 라벨링 프로세스

1. **CSV 파일 열기** (`evaluation_dataset.csv`)
2. **리뷰 읽기** (`review_text` 컬럼)
3. **카테고리 선택** → `manual_label` 컬럼에 입력
4. **메모** (선택사항) → `notes` 컬럼에 입력
5. **저장**

### 6. 품질 체크

10개마다 확인:
- 일관성 있게 라벨링했는지
- 카테고리 목록 참고했는지
- 핵심 문제에 집중했는지

## 예시

| review_text | manual_label | notes |
|-------------|--------------|-------|
| "Package took 3 weeks to arrive" | delivery_delay | 명확함 |
| "Wrong color received, wanted blue" | wrong_item | 색상 잘못됨 |
| "Cheap material, broke after 2 days" | poor_quality | 품질 문제 |
| "Delivery was slow AND item was damaged" | damaged_packaging | 둘 다 언급하지만 파손이 더 심각 |

## 시간 관리

- 100개 라벨링 예상 시간: **2-3시간**
- 한 번에 다 하지 말고 30개씩 나눠서 진행
- 집중력 떨어지면 쉬었다가 다시

## 완료 후

라벨링 완료하면 저에게 알려주세요!
→ `evaluate.py` 스크립트로 AI 정확도 측정 시작
