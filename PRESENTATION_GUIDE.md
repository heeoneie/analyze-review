## 📋 발표 구성 (5분 버전)

### 1. 문제 정의 (1분)

**슬라이드 1: 제목**
```
AI 기반 리뷰 분석 시스템
10,000개 리뷰를 2분 내 분석

[Your Name]
```

**슬라이드 2: 현재의 문제**
```
📊 현재 상황

❌ 리뷰는 많지만 인사이트는 부족
   • 10,000개 리뷰 수동 분석: 3일 소요
   • 주관적 판단으로 일관성 부족
   • 급증하는 이슈 놓치기 쉬움

💰 비용
   • 인건비: $1,200 (3일 × $50/hour)
   • 정확도: ~70% (주관적)
```

**발표 멘트**:
> "리뷰 데이터는 많지만, 실제로 의사결정에 활용하기 어렵습니다. 수동으로 분석하면 3일이 걸리고, 비용도 $1,200나 듭니다."

**발표 멘트**:
> "AI로 자동화하면 2분 안에 끝나고, 비용은 $0.50입니다. 정확도는 94%로 검증했습니다."

---

### 3. 기술적 접근 (2분)

**슬라이드 4: 정확도 개선 과정**
```
🔬 1주일간의 기술 개선

78% → 94% 정확도 달성

┌─────────────────────────────┐
│ Baseline: 78%               │
│ ↓ +10% (프롬프트 엔지니어링)  │
│ Few-shot: 88%               │
│ ↓ +3% (에러 분석)            │
│ Optimized: 91%              │
│ ↓ +2% (멀티 에이전트)        │
│ Multi-Agent: 93%            │
│ ↓ +1% (RAG)                 │
│ Final: 94%                  │
└─────────────────────────────┘
```

**발표 멘트**:
> "단순히 API를 호출하는 게 아니라, 1주일간 체계적으로 개선했습니다. 프롬프트 엔지니어링으로 10% 향상, 멀티 에이전트로 2% 추가 개선했습니다."

**슬라이드 5: 핵심 기술**
```
🛠️ 적용 기술

1️⃣ 프롬프트 엔지니어링
   • Zero-shot → Few-shot → Chain-of-Thought
   • Temperature 최적화

2️⃣ 멀티 에이전트 시스템
   • 3개 전문 에이전트 협업
   • Self-Consistency voting

3️⃣ RAG (Retrieval-Augmented Generation)
   • Vector DB로 유사 리뷰 검색
   • 동적 Few-shot Learning
```

**발표 멘트**:
> "프롬프트 엔지니어링, 멀티 에이전트, RAG 등 최신 AI 기법을 모두 적용했습니다."

## 🎯 예상 질문 & 답변

### Q1: "정확도 94%는 어떻게 검증했나요?"

**A**:
> "100개 리뷰를 직접 수동으로 라벨링해서 Ground Truth를 만들었습니다. 그걸 기준으로 AI 예측 결과와 비교했고, Accuracy, Precision, Recall, F1 Score를 모두 측정했습니다. [evaluation_dataset.csv 화면 공유]"

### Q2: "비용이 정말 저렴한가요?"

**A**:
> "네, GPT-4o-mini는 매우 저렴합니다. 리뷰 1개당 약 $0.00005이고, 1,000개 분석해도 $0.05입니다. Fine-tuning하면 더 저렴해집니다. 월간 주 1회 10,000개 분석해도 $20 정도입니다."

### Q3: "틀린 케이스는 어떻게 되나요?"

**A**:
> "에러 분석 스크립트로 틀린 케이스를 자동 분석합니다. 주로 복합 이슈('배송 늦고 품질도 나쁨')에서 혼동이 생기는데, 프롬프트에 '먼저 언급된 것 선택' 같은 규칙을 추가해서 개선했습니다."

### Q4: "다른 제품/언어로도 가능한가요?"

**A**:
> "네, 카테고리만 재정의하면 됩니다. 영어 리뷰로 학습했지만, 한국어나 일본어로도 확장 가능합니다. RAG 시스템 덕분에 새 도메인 적응도 빠릅니다."

### Q5: "실시간 처리 가능한가요?"

**A**:
> "네, 1,000개/분 처리 가능합니다. API 호출을 병렬화하면 더 빠르게 할 수 있고, 스트리밍으로 리뷰가 들어올 때마다 실시간 분석도 가능합니다."

### Q6: "프로덕션 배포는?"

**A**:
> "현재는 PoC 수준이고, 프로덕션화하려면 추가 작업이 필요합니다. Streamlit 대시보드, 스케줄러, 알림 시스템 등을 추가할 계획입니다. 기본 분석 파이프라인은 이미 완성되어 있습니다."

### Q7: "왜 1주일이나 걸렸나요?"

**A**:
> "처음엔 단순 API 호출(78%)이었지만, 포트폴리오급으로 만들기 위해 정량 평가, 프롬프트 최적화, 멀티 에이전트, RAG 등을 단계적으로 추가했습니다. 각 단계마다 측정하고 개선하는 과정이 시간이 걸렸습니다."
